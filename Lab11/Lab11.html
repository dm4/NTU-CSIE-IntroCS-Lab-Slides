Lab11 Version Control System

Version Control System (VCS)

  • Backup your code each version

  • Log for modification each version

  • Collaborate with people

  • Make your developement easier

Local VCS

  • like cp -r each version

Centralized VCS

  • CVS, Subversion(SVN)

Centralized VCS

  • Checkout from server
  • Commit to server
  • Doing anything with server

Distributed VCS

  • Git, Mercurial(Hg)

Distributed VCS

  • Commit without connecting to server
  • View log without connecting to server
  • Commit individually, merge when pushing/pulling

Git

Git do not

  • Git do NOT just keep the differeces of files each version

Git DO

  • Git DO keep all the snapshots of files each version

Work flow

  • Modify, Stage, Commit

Create a Git repository

  • Create new

    $ mkdir project
$ cd project
$ git init
$ vim README
$ git add README
$ git commit            # type commit log in editor

Create a Git repository

  • Create from existed repository

    $ git clone /path/to/existed/repository
