Google App Engine

  • GAE provides hardwares of server.
  • GAE also provides some libraries for developers to access these hardwares.
  • Developers can concentrate on the software part.

Two HTTP Methods

The ways you can upload and download files from server.

  • GET

    • Pass arguments by adding key-value pairs directly after website url.

      http://www.google.com.tw/search?ie=UTF-8&q=123
http://www.google.com.tw/search?ie=UTF-8&q=abc
    • Easy to use, only need to modify url.
    • Usually use GET when you are querying a server.
    • Because url has the maximum length, the data you can pass also have limitation.

Two HTTP Methods (cont.)

  • POST
    • Pass arguments by key-value pairs which are stored with the data you send.
    • It doesn't have length limitation, so it suits for sending files whose size is large. eg. photos
    • Usually, we will pass our username and password by this method, because everyone can catch your url. It's not good to pass secret information by GET.

Create a GAE Project

  1. Create a working directory.

    r00922071@linux15 [~] mkdir {your_dir}

  2. Create a app.yaml file

    application: helloworld
version: 1
runtime: python
api_version: 1
handlers:
- url: /.*
  script: helloworld.py
  3. Now you can create helloworld.py file and develope your apps there!

Google App Engine

Take a Look of Your App

GAE provides a dev server for developers to debug. - cd to your working directory.

    dev_appserver.py .

- Now open your browser and link to 

    http://localhost:8080

- Default port number is 8080. You can add the arguments to change the port on which your dev server binds.

    r00922071@linux15 [~] dev_appserver.py . --port=8081  //bind on port 8081

About Port

  • You can launch multiple dev servers on your computer, each one binds on the different port.
  • Each computer may have multiple servers running on it at the same time. Port number let computer know which server you'd like to connect.
